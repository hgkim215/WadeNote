# WadeNote 설계 문서

- 작성일: 2026-06-22
- 상태: 승인됨 (구현 계획 수립 전)

## 1. 개요

WadeNote는 민감한 개인정보를 **유형별 템플릿**으로 구조화 저장하고, **Face ID**로 잠그며, **iCloud(CloudKit)**로 모든 기기에 자동 동기화·복원하는 iOS 네이티브 앱이다. 참고 모델은 "비밀번호 뭐였지?" 류의 개인정보 보관 앱이며, 차별점은 **분류와 가독성**이다.

### 목표
- 외우기 싫은 민감한 값들(계정/비밀번호, 카드/은행, 신분증, 보안 메모)을 한눈에 읽히게 구조화 저장.
- 기기를 바꾸거나 앱을 재설치해도 iCloud에서 자동 복원.
- 철저한 보안: 앱 잠금 + 클라우드 암호화 + 사진 추가 암호화.

### 비목표 (v1)
- 비밀번호 생성기, iOS 자동완성(AutoFill) 제공자, 제로-지식 마스터 비밀번호, 폴더, 태그, 만료 알림 — 모두 v2 이후.
- Android·웹 — iOS 전용.

## 2. 데이터 모델

**아이템 = 하나의 노트(카드)**. 각 아이템은 하나의 **유형(템플릿)**에 속하고, 유형마다 정해진 필드를 가진다.

| 유형 | 기본 필드 |
|------|-----------|
| 로그인 | 서비스명, 아이디, 비밀번호, URL, 메모 |
| 카드/은행 | 카드/계좌명, 번호, 유효기간, CVC, 비밀번호, 메모 |
| 신분증 | 종류, 번호, 발급일, 만료일, 메모 |
| 보안 메모 | 제목, 본문(자유 텍스트) |

### 필드 속성
- 각 필드 = `라벨 + 값 + 타입`.
- 타입: `텍스트 / 비밀값 / URL / 날짜`.
- **비밀값** 타입은 기본 가려짐(••••), 탭하면 보기/복사.
- 모든 필드는 "빠른 복사" 대상.

### 공통 메타데이터
- 제목, 유형, 즐겨찾기 여부, 첨부 사진들, 생성/수정일.
- **사용자 정의 필드 추가** 가능.

### 설계 의도
자유 메모의 유연함은 "보안 메모" 유형으로 남기되, 나머지는 구조화해서 한눈에 읽히고 빠르게 복사되게 한다.

## 3. 아키텍처

### 기술 스택
- **SwiftUI** — UI
- **SwiftData** — 영속성 및 모델 정의 (`@Model`)
- **SwiftData ↔ CloudKit 통합** — `ModelConfiguration`에 CloudKit private database 지정 → 저장 시 자동 iCloud 동기화·복원. 별도 동기화 코드 최소화.
- **LocalAuthentication** — Face ID / Touch ID / 패스코드 폴백
- **CryptoKit** — 사진 첨부 암호화
- **Keychain** (Secure Enclave 보호) — 암호화 키 보관

### 모듈 경계
각 모듈은 독립적으로 이해·테스트 가능하도록 하나의 명확한 책임을 가진다.

| 모듈 | 역할 | 의존 |
|------|------|------|
| `Models` | 아이템·필드·유형 정의 (SwiftData `@Model`) | 없음 |
| `Templates` | 유형별 기본 필드 정의 (데이터로 기술) | Models |
| `Store` | CRUD·검색·즐겨찾기·CloudKit 설정 | Models |
| `AttachmentStore` | 사진 암호화 저장/복호화 | CryptoKit |
| `AppLock` | Face ID 잠금·자동잠금 타이머 | LocalAuthentication |
| `Clipboard` | 복사 + N초 후 자동 삭제 | 없음 |
| UI (화면들) | 목록·상세·편집·잠금 화면 | 위 전부 |

## 4. 보안 모델

기기 잠금 + iCloud 의존 방식. (마스터 비밀번호 분실 위험 없음.)

- **앱 진입**: Face ID 필수. 백그라운드 진입/일정 시간 후 자동 재잠금. 잠금 중엔 앱 스위처 미리보기도 가린다.
- **저장 데이터(텍스트 필드)**: CloudKit private database — 전송·저장 시 Apple이 암호화. 사용자에게 **고급 데이터 보호(ADP) 활성화 안내** → 켜면 사실상 E2E.
- **사진 첨부**: 원본을 그대로 두지 않고 **CryptoKit으로 암호화**, 키는 **Keychain**(Secure Enclave)에 보관. (실물 이미지가 텍스트보다 민감하므로 한 겹 더.)
- **클립보드**: 빠른 복사 시 민감값은 N초 후 자동 삭제. 필요 화면 스크린샷/캡처 가림.

### 비대칭 결정 (승인됨)
사진은 CryptoKit으로 추가 암호화하고, 텍스트는 CloudKit + ADP에 의존한다.

### 확장 여지
나중에 제로-지식이 필요하면 `Store` 앞단에 "마스터키로 필드값 암호화" 레이어를 끼울 수 있게 인터페이스만 열어둔다. v1에서는 구현하지 않는다 (YAGNI).

## 5. 화면 흐름

1. **잠금 화면** — 앱 진입/재진입 시 Face ID. 실패 시 패스코드 폴백.
2. **목록(홈)** — 상단 검색바 + 즐겨찾기 섹션, 그 아래 유형별 그룹 또는 전체 리스트. 각 행은 제목·유형 아이콘·즐겨찾기 별. 행에서 바로 "대표 필드 빠른 복사".
3. **상세** — 구조화된 카드 뷰. 비밀값은 ••••로 가려짐, 탭하면 보기/복사. 각 필드 옆 복사 버튼. 첨부 사진 썸네일.
4. **편집/추가** — 유형 선택 → 템플릿 필드 자동 생성 → 값 입력. 사용자 정의 필드 추가. 사진 첨부.

### 상호작용 디테일
- **빠른 복사**: 탭 → 복사됨 토스트 → N초 후 클립보드 자동 삭제.
- **검색**: 제목·유형·필드 라벨 대상 (필드 *값*은 보안상 검색 제외 검토).
- **즐겨찾기**: 별 토글 → 홈 상단 고정.

## 6. v1 범위

확정:
- ✅ 유형별 템플릿 4종 + 사용자 정의 필드
- ✅ Face ID 잠금 + 자동 재잠금
- ✅ CloudKit 동기화/복원 + ADP 안내
- ✅ 필드 빠른 복사 (자동 삭제)
- ✅ 검색 / 즐겨찾기
- ✅ 사진 첨부 (암호화)

v2 이후로 미룸: 비밀번호 생성기, iOS 자동완성(AutoFill) 제공자, 제로-지식 마스터 비밀번호, 폴더, 태그, 만료 알림.

## 7. 테스트 전략

- 단위 테스트: `Templates`, `Store`, `Clipboard`, `AttachmentStore`(암복호화 라운드트립).
- 통합/수동 테스트: `AppLock`, CloudKit 동기화·복원.
